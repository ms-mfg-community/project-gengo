p11. [] Can a subnet size of 10.100.0.0/24 fit inside a vnet address space of 10.100.0.0/22 ?
r11. []
Yes, a subnet with the address block of 10.100.0.0/24 can fit inside a Virtual Network (VNet) with the address space of 10.100.0.0/22.
Here's why:
•	The address space 10.100.0.0/22 covers the IP address range from 10.100.0.0 to 10.100.3.255.
•	The subnet 10.100.0.0/24 covers the IP address range from 10.100.0.0 to 10.100.0.255.
As you can see, the range of the /24 subnet falls entirely within the range of the /22 VNet. Therefore, the subnet can fit inside the VNet.

p12. [] /generate deploy the hub resource group in file  #hubResources.bicep
r12. []
targetScope = 'resourceGroup'

param hub object

// Deploy a virtual network in the hub resource group
resource hubVnet 'Microsoft.Network/virtualNetworks@2021-02-01' = {
  name: hub.name
  location: resourceGroup().location
  properties: {
    addressSpace: {
      addressPrefixes: hub.addressPrefixes
    }
    subnets: [
      for subnet in hub.subnets: {
        name: subnet.name
        properties: {
          addressPrefix: subnet.addressPrefix
        }
      }
    ]
  }
}

// Output the hub object
output hubObject object = {
  name: hubVnet.name
  location: hubVnet.location
  addressSpace: hubVnet.properties.addressSpace
  subnets: hubVnet.properties.subnets
}
