[] To import a CSV file into an Azure-hosted PostgreSQL database using `psql`, you can follow these steps:
[] 1. Assume you have a file named `ghc-skills-matrix-demo-catalog.csv` that is already populated with values.
[] 2. [prompt] create an Azure keyvault instance to save the postgres user secret:
    az keyvault secret set --vault-name "psql-kvt-01" --name "db-password" --value "mypassword"
[] 2. [prompt] Connect to postgres host pfs-sql-01.postgres.database.azure.com on port 5432 with username ztmadmin for the database ghc_prompts and prompt for password
[] 3. **Connect to the PostgreSQL database**: ```sh psql -h pfs-sql-01.postgres.database.azure.com -U ztmadmin -d ghc_prompts -W ```
[] 4. create enum types:
```psql
create type prompt_type_enum as enum ('chat','inline');
create type shot_type_enum as enum ('zero','one','multiple');
create type test_type_enum as enum ('na','unit','integration','performance','load','resiliency','security');
create type ide_type_enum as enum ('vs_code','visual_studio','jetbrains_suite','vim_neovim','xcode','none','other');

create table demo_catalog (
    index int primary key,
    category varchar(20),
    sub_category varchar(20),
    language varchar(20),
    role varchar(50),
    person varchar (50),
    ide_type ide_type_enum not null,
    prompt_type prompt_type_enum not null,
    shot_type shot_type_enum not null,
    is_test boolean not null,
    test_type test_type_enum not null,
    epoch int check (epoch >= 1 AND epoch <= 10),
    confidence_percent int check (confidence_percent >= 10 and confidence_percent <= 100),
    scenario varchar(255),
    github_org varchar(20),
    reference varchar(255),
    notes_feedback varchar(255)
);

exit 
```
