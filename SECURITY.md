# Security Policy

## <!-- BEGIN MICROSOFT SECURITY.MD V0.0.9 BLOCK -->

## Overview

We take the security of our software products and services seriously. This includes all source code repositories managed through our GitHub organization. We are committed to promptly addressing security vulnerabilities and maintaining the safety of our users, contributors, and ecosystem.

## Reporting a Vulnerability

If you discover a potential security vulnerability, please follow these guidelines:

- **Do NOT report security vulnerabilities through public GitHub issues, discussions, or pull requests.**

- Email your findings to: [prestopa@microsoft.com](mailto:prestopa@microsoft.com).

- If possible, encrypt your message using our PGP key, available from the [Microsoft Security PGP Key](https://www.microsoft.com/en-us/msrc/pgp-key-msrc).

## Response Expectation

You should receive a response within 24 hours. If you do not receive a timely reply, please follow up to ensure your message was received.

### Please Include (if possible)

- Type of issue (e.g., buffer overflow, SQL injection, cross-site scripting, etc.)

- Full paths of source file(s) related to the issue

- Location of the affected source code (Git tag/branch/commit or direct URL)

- Any special configuration required to reproduce the issue

- Step-by-step instructions to reproduce the issue

- Proof-of-concept or exploit code (if available)

- Impact assessment (how an attacker might exploit the issue)

- Any known mitigations or workarounds

Providing as much detail as possible will help us triage your report quickly and accurately.

## Responsible Disclosure

We ask that you:

- Give us a reasonable time to investigate and address an issue before disclosing it publicly.

- Act in good faith to avoid privacy violations, destruction of data, and interruption or degradation of our services.

- Only interact with your own accounts or test accounts for security research.

We are committed to coordinating with you to remediate validated vulnerabilities in a timely manner and will keep you updated on our progress.

## Remediation Timeline

- **Acknowledgment:** We aim to acknowledge all reports within 24 hours.

- **Assessment:** Triage and initial assessment will be completed within 5 business days.

- **Resolution:** We target resolution or mitigation within 90 days of acknowledgment, depending on severity and complexity. If a fix requires more time, we will communicate status updates.

## Security Updates

For critical security updates affecting users or contributors, we will:

- Issue a new release with appropriate release notes

- Announce the update in the repository’s README and release channels

## Preferred Language

All communications should be in English.

## Acknowledgments

We appreciate the contributions of security researchers and community members who responsibly disclose vulnerabilities. With your permission, we may credit you in security advisories or release notes.
